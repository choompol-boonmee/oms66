3. Data Center Configuration This clause presents a conceptual high-level overview of each TDMS data center from a server configuration perspective. 3.1 General Concept In general, the TDMS platform at each data center shall consist of virtual machines configured on several physical servers and interconnected via redundant Virtual Local Area Networks (VLANs) suitably segregated and secure to meet the System’s functional requirements. Unless otherwise required, Open rather than OEM licensing shall apply. A conceptual configuration based on server virtualization is illustrated in Exhibit 3-1, noting that the exhibit does not identify servers such as those as may be required to support the User Interface facilities located remotely at the SMC and ADDC control rooms, nor does it identify a configuration in which, for example, the Virtualized Simulator Environment (SIE) may be in a separate security zone (as determined by the Contractor’s best security practices) and suitably interfaced, as described elsewhere in these Technical Specifications, to control-room (Dispatcher) workstations as well as those in the Authority’s training center. 3.1.1 Production Environment (PDE) 3.1.2 Process Environment (PCE) 3.1.3 Pre-Production Environment (PPE) 3.1.3.1 DVS Functions 3.1.3.2 QAS Functions 3.1.4 DMZ Environment (DMZE) 3.1.5 System Management Environment (SME) 3.1.6 Simulator Environment (SIE)
